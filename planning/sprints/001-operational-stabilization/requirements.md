# Sprint 001 Requirements: Operational Stabilization

## Purpose
Adopt the 120x operating structure inside the existing Meeting Tool repo so product direction, architecture, decisions, risks, questions, and handoffs live with the codebase.

## Target Users
- Project owner who needs a durable source of truth.
- Codex working on future PRs.
- Future builders who need concise project context before implementation.

## In Scope
- Create/update `AGENTS.md` and `CODEX.md`.
- Create planning files under `planning/`.
- Create sprint handoff files under `planning/sprints/001-operational-stabilization/`.
- Create documentation files under `docs/`.
- Capture current project facts, decisions, risks, open questions, branch strategy, architecture, data model, validation, and permissions planning.

## Out of Scope
- Production application code changes.
- App behavior changes.
- Moving existing app files.
- Cloud implementation.
- Authentication or permission enforcement.
- Supabase schema migrations.
- Creating `src/`, `tests/`, `scripts/`, `samples/`, or `references/`.

## Assumptions
- The existing Next.js app structure remains the production app structure.
- Phase 1 remains focused on local operational usability and stability.
